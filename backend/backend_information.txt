Deep Saga 3.0 Backend Overview
==============================

Current Backend Shape
---------------------

The active backend is now intentionally small:

- app.js mounts the Express app.
- routes/auth.js handles account registration and login.
- routes/rpg.js handles the chat-based monster reincarnation game.
- services/monsterRpgEngine.js resolves RPG state, narration outcomes, choices, stats, skills, inventory, evolution, and floor progress.

Removed Legacy Surface
----------------------

The old dungeon/play/story/rebirth/ascension/player routers are no longer part of the app.
The old support services for those routes were also removed.

Active API Routes
-----------------

Authentication:

- POST /api/auth/register
- POST /api/auth/login

RPG:

- GET /api/rpg/state
- POST /api/rpg/start
- POST /api/rpg/action

Database Migrations
-------------------

Run migrations in order:

1. migrations/001_initial_schema.sql
2. migrations/002_monster_reincarnation_rpg.sql
3. migrations/003_single_rpg_flow_and_10_floors.sql
4. migrations/005_remove_legacy_unused_tables.sql

Notes:

- 001 still creates early legacy tables because it already existed.
- 005 drops the legacy tables no longer used by the chat-based RPG.
- Do not edit old migrations. Add a new numbered migration for future schema changes.
